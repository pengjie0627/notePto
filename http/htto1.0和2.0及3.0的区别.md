HTTP/1.0
特点：

1、无连接：每次请求/响应都需要建立一个新的 TCP 连接，导致效率低下。
2、无状态：每个请求都是独立的，服务器不会保存客户端的状态信息。
3、不支持持久连接：每次请求完成后连接就会关闭。
4、不支持管道化：客户端必须等待上一个请求的响应完成才能发送下一个请求。
5、请求头不完整：缺乏一些常用的请求头字段，如 Host，需要客户端和服务器之间约定。
优点：

简单易实现。
缺点：

效率低下，每次请求都需要建立新的连接。
不支持并发请求，影响页面加载速度。
HTTP/1.1
特点：

1、持久连接：允许在一个 TCP 连接上处理多个请求，减少了连接建立的时间开销。
2、管道化：客户端可以在收到前一个请求的响应之前发送多个请求，提高效率。
3、请求头更完整：增加了 Host 头字段，解决了虚拟主机的问题。
4、缓存机制：支持更多的缓存控制头字段，如 Cache-Control 和 ETag。
5、分块传输编码：服务器可以分块发送响应体，客户端可以逐步处理数据，而不需要等待整个响应体下载完毕。
优点：

提高了连接的复用性，减少了延迟。
支持并发请求，提高了页面加载速度。
更丰富的请求头和响应头字段，提供了更好的控制和优化。
缺点：

仍然存在队头阻塞问题，即在一个连接上的请求必须按顺序处理，后面的请求会被前面的请求阻塞。
HTTP/2
特点：

1、多路复用：在一个连接上可以同时发送多个请求和响应，解决了 HTTP/1.1 的队头阻塞问题。
2、头部压缩：使用 HPACK 压缩算法减少头部信息的传输量，提高传输效率。
3、服务器推送：服务器可以主动推送资源到客户端的缓存中，减少后续请求的延迟。
4、二进制格式：使用二进制格式传输数据，而不是文本格式，提高了解析效率。
5、优先级：客户端可以为不同的请求设置优先级，服务器可以根据优先级调度响应。
优点：

显著提高了页面加载速度和用户体验。
减少了网络带宽的消耗。
提供了更灵活的资源管理和调度机制。
缺点：

实现复杂度增加。
需要客户端和服务器都支持 HTTP/2。
HTTP/3
特点：

1、基于 QUIC 协议：QUIC 是一个基于 UDP 的传输层协议，旨在解决 TCP 的某些固有问题，如连接建立的延迟和丢包重传的效率。
2、连接迁移：当客户端的 IP 地址或端口发生变化时，QUIC 可以保持连接不中断，适用于移动设备在网络切换时的场景。
3、无队头阻塞：由于 QUIC 的特性，HTTP/3 可以更好地避免队头阻塞问题。
4、更快的连接建立：QUIC 使用 0-RTT（Zero Round Trip Time）握手，减少了连接建立的时间。
5、更强的安全性：QUIC 默认使用 TLS 1.3 进行加密，提供更高的安全性。
优点：

进一步提高了网络性能和可靠性。
提供了更好的用户体验，特别是在移动网络环境下。
加强了安全性和隐私保护。
缺点：

实现和部署更加复杂。
需要客户端和服务器都支持 QUIC 和 HTTP/3。
总结
HTTP/1.0：简单但效率低，每次请求都需要建立新的连接。
HTTP/1.1：引入了持久连接和管道化，提高了效率，但仍存在队头阻塞问题。
HTTP/2：通过多路复用和头部压缩等技术，显著提高了性能。
HTTP/3：基于 QUIC 协议，进一步提高了网络性能和可靠性，特别适合移动网络环境。
