步骤：
```
1、打开谷歌设置，进入商店，下载lighthouse插件并安装
2、打开浏览器控制台，控制台中会有lighthouse选项，选项下有下面几大分类：
【1】Performance：该选项用于评估Web应用程序的性能。它会分析页面加载时间、资源大小、缓存策略等因素，并给出相应的建议和优化方案。
【2】Accessibility：该选项用于评估Web应用程序的可访问性。它会检查页面是否符合WCAG（Web Content Accessibility Guidelines）标准，并给出相应的建议和优化方案。
【3】Best practices：该选项用于评估Web应用程序是否符合最佳实践。它会检查页面是否存在常见的问题，例如安全问题、SEO问题等，并给出相应的建议和优化方案。
【4】SEO：该选项用于评估Web应用程序的搜索引擎优化情况。它会检查页面是否符合搜索引擎的要求，例如页面标题、meta标签等，并给出相应的建议和优化方案。
【5】Progressive Web App：选项用于评估Web应用程序是否符合渐进式Web应用程序（PWA）的标准。它会检查页面是否具有离线访问、添加到主屏幕等功能，并给出相应的建议和优化方案。
3、常规情况下，我们重点关注第一个Performance，我们只勾选它，然后勾选进行web分析还是mobile分析，然后点击分析按钮
4、分析出结果后，我们会看到分析结果：
【1】FCP（First Contentful Paint）：白屏时间，第一次有内容呈现在用户屏幕上的时间点。
【2】TTI（Time to Interactive）：第一次可流畅交互的时间。
【3】SI（Speed Index）：是一个衡量Web页面加载速度的指标，它表示页面在加载过程中的视觉稳定性。Speed Index越低，表示页面加载速度越快，用户体验越好，视觉稳定性得分是根据页面中元素的位置和大小变化来计算的，变化越大得分越高，变化越小得分越低。
【4】TBT（Total Blocking Time）：它是指在FCP和TTI之间，所有超过50毫秒的任务时间总和，以毫秒为单位。
【5】LCP（Largest Contentful Paint）：指页面渲染出最大文本或图片的时间。
【6】CLS（Cumulative Layout Shift）：布局偏移情况，当页面中的元素位置发生变化时，浏览器需要重新计算页面的布局，从而增加页面加载时间。如果页面中的元素位置变化过于频繁，会导致页面加载时间过长，影响用户体验和SEO排名。
【7】同时，我们还能看到静态资源的利用率情况：点击View Treemap
5、接下来我么可以看下Opportunities这个节点下面的优化建议信息，然后按照提示信息进行处理即可
6、消除页面非关键资源
【1】chrome浏览器，按ctrl+shift+p打开命令菜单，输入coverage命令，底部出现一个新的控制台，然后刷新页面，可以看到页面所需要的资源
【2】chrome浏览器，ctrl+shift+p再次打开命令菜单，输入blocking命令，左下角点击 + 号，添加阻塞规则（就是把文件路径作为阻塞规则添加进去，），这样该文件就会被禁止。如果金之后页面白屏则是必须资源，否则不是必须资源，可以异步加载（比如增加defer标识）
7、利用Chrome的开发者工具，打开控制台，选择Performance ，然后选择左上角的start按钮进行，查看哪些js运行时间长，然后找到该js进行分析调整，这样就可以让页面性能很高
```
